## 📋 一、自动化工作流

```
┌────────────────────────────────────────────────────────────────────────┐
│                         自动化开发流程                                   │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   1️⃣ 需求分析                                                            │
│       └→ 更新 REQUIREMENTS.md                                          │
│                                                                        │
│   2️⃣ 任务拆分                                                            │
│       ├→ 前端任务 → Agent-A (前端+桌面)                                │
│       └→ 后端任务 → Agent-B (API+数据)                                 │
│                                                                        │
│   3️⃣ 开发执行 (Sub-agent 并行)                                          │
│       ├→ Agent-A: 写代码 → 提交PR                                      │
│       └→ Agent-B: 写代码 → 提交PR                                      │
│                                                                        │
│   4️⃣ 代码审核 (自动)                                                    │
│       ├→ 验证功能正常                                                  │
│       ├→ 检查代码质量                                                  │
│       └→ 合并PR / 打回修改                                              │
│                                                                        │
│   5️⃣ 持续迭代 (不等待)                                                  │
│       └→ 完成一个Issue → 继续下一个 → 更新PROCESS.md                  │
│                                                                        │
│   6️⃣ 集成测试 (全部完成后)                                              │
│       └→ 启动所有服务 → 全面测试 → 记录Bug                            │
│                                                                        │
│   7️⃣ Bug修复循环                                                        │
│       └→ 更新需求 → 拆分任务 → 执行 → 测试                            │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

## 🔧 二、Sub-agent 协作

### Agent-A: 前端 + 桌面应用
| 任务类型 | 说明 |
|----------|------|
| Electron 集成 | 桌面应用主进程 |
| UI 开发 | 流量监控、对话面板 |
| 组件开发 | 现有功能完善 |
| 打包发布 | exe 构建 |

### Agent-B: 后端 + 数据
| 任务类型 | 说明 |
|----------|------|
| API 开发 | 新功能接口 |
| 数据连接 | OpenClaw 数据对接 |
| Bug 修复 | 后端问题 |
| 数据库 | 数据存储 |

## 🔄 四、工作流程细则

### 4.1 创建 Issue
```bash
python skills/github/scripts/create_issue.py <owner> <repo> <title> <body> <labels>
```

### 4.2 分配任务
```python
sessions_spawn(
    task="<Issue 详细描述>",
    agentId="fullstack-dev",
    mode="run",
    timeoutSeconds=3600
)
```

### 4.3 代码提交
```bash
git checkout -b feature/<issue-id>-<描述>
git commit -m "<类型>: <描述> (#<issue-id>)"
git push origin feature/<issue-id>-<描述>
```

### 4.4 创建 PR
```bash
python skills/github/scripts/create_pr.py <owner> <repo> <title> <head> <base> <body>
```

### 4.5 审核合并
1. 检查代码变更
2. 验证功能正常
3. 合并到 develop 分支
4. 关闭 Issue

### 4.6 更新 PROCESS.md
每次完成 Issue 后更新本文档！

## 🔔 六、关键规则

| 规则 | 说明 |
|------|------|
| 不等待 | 完成一个Issue → 继续下一个 |
| 持续迭代 | 无需等待整个阶段完成 |
| 更新文档 | 每次完成Issue更新PROCESS.md |
| 自动审核 | 验证通过自动合并 |
| 集成测试 | 全部完成后全面测试 |
